# План разработки приложения для изучения языков (MVP)

Ниже приведен план разработки PWA-приложения для изучения языков, разбитый на этапы с задачами, описанием, ответственными, сроками и зависимостями. Срок для MVP: 4-6 недель (при одном разработчике или небольшой команде).

## Этап 1: Подготовка и дизайн (1 неделя)

| Задача        | Описание                                                                                                                                                                                                                                                                                                                                                                   | Ответственный      | Срок   | Зависимости |
| ------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------ | ------ | ----------- |
| Дизайн UI/UX  | Создать макеты (Figma/Sketch): главная страница, создание словаря, добавление слов (с автодополнением из БД, API картинок/переводов), магазин (поиск по языкам/темам), превью (название, описание, языки), дашборд (мои/купленные словари с иконками звезда/монета). Учесть мобильный вид для PWA.                                                                         | Дизайнер/Вы        | День 3 | Нет         |
| Модели БД     | Определить модели Django: `User` (расширение AbstractUser), `Dictionary` (название, описание, исходный*язык, целевой*язык, владелец, цена, временный*доступ, длительность*в*днях), `Word` (слово, перевод, URL*картинки, пример, FK словаря), `Card` (глобальная БД слов для избежания дубликатов), `Purchase` (пользователь, словарь, тип_доступа: постоянный/временный). | Бэкенд-разработчик | День 5 | Дизайн      |
| API-эндпоинты | Спланировать эндпоинты DRF: `/auth/register/login`, `/dictionaries/create/list/mine/purchased`, `/words/add` (с поиском в глобальной БД), `/store/search` (фильтры по языкам/темам), `/payments/buy` (вебхук Stripe), `/withdraw` (вывод с комиссией 30%).                                                                                                                 | Бэкенд-разработчик | День 7 | Модели      |
